Troubleshooting Guide

Common Authentication Problems
  • Invalid API Keys - Ensure your API key and secret are copied correctly from your broker's dashboard
  • Expired Tokens - OAuth tokens may expire and need to be refreshed
  • Incorrect Environment - Make sure you're using the right keys for paper vs. live trading
  • Account Restrictions - Your account may have restrictions or need additional verification

Network Connectivity Troubleshooting
  • DNS Resolution - If you can't resolve hostnames, check your DNS settings or try using IP addresses
  • SSL/TLS Errors - Certificate validation issues may indicate a misconfigured system or security software
  • Firewalls/Proxy - Your network may be blocking financial API connections
  • API Downtime - Check broker status pages for any reported outages
You can use the "Test Specific Endpoints" feature to isolate which particular API services are causing issues.

Optimization Tips
  • Cache data when possible to reduce API calls
  • Implement exponential backoff for retries
  • Use websocket connections for real-time data instead of polling
  • Batch requests when the API supports it

About Simulation Mode

Simulation mode allows you to use the trading bot without a live API connection. This is useful for:

  • Testing the interface without real API credentials
  • Developing and testing new strategies
  • Learning how the platform works without risk
  • Using the bot during API outages
Important: Simulation mode uses randomly generated data and does not reflect real market conditions or account balances.
Enabling Simulation Mode

To enable simulation mode, go to Settings and check the "Force Simulation Mode" option. This will bypass all API connection attempts and use simulated data.
